# DSA Patterns & Interview Preparation

A structured collection of Data Structures & Algorithms problems  
organized by **patterns**, not difficulty.

---

## 🎯 Philosophy

This repository focuses on:

- Pattern recognition over memorization
- Clean, documented solutions
- Clear complexity analysis
- Scalable learning structure

Instead of organizing problems as Easy/Medium/Hard,
they are grouped by **problem-solving patterns**.

Because interviews test patterns — not problem titles.

---

## 🧠 Core Learning Strategy

For every problem:

1. Identify the pattern
2. Extract the key insight
3. Document complexity
4. Compare alternative approaches
5. Generalize the technique

Each problem folder contains:
- `solution.py`
- A structured explanation (README.md)

## 📌 How to Use This Repo

If you are revising:

1. Pick a pattern.
2. Solve 5–10 problems in that category.
3. Focus on recognizing structural similarities.
4. Re-implement from memory.

If you are preparing for interviews:

- Start with Arrays & Hashing.
- Then Sliding Window.
- Then Binary Search.
- Then Trees and Graphs.
- Leave DP for later stages.
